## Issue 1: [Phase 1] Expoプロジェクトの初期化と基本環境構築

### タイトル
`[Phase 1] Expoプロジェクトの初期化と基本環境構築`

### 本文
```markdown
### 概要
「パック＆メモ」のReact Native (Expo) 開発を開始するために、Expo Router ベースの TypeScript プロジェクトを新規生成し、必要な依存ライブラリを導入します。

### タスクリスト
- [ ] `npx create-expo-app@latest` を用いて、TypeScript / Expo Router 構成でプロジェクトを初期化
- [ ] 必要な基本パッケージのインストール
  - `expo-sqlite` (データベース用)
  - `expo-image-picker` (カメラ・フォトピッカー用)
  - `expo-file-system` (写真などのファイル管理用)
  - `react-native-safe-area-context` (Android 15のEdge-to-Edge対応用)
- [ ] プロジェクトの不要なデフォルトファイル（テンプレート画面等）のクリーンアップ
- [ ] 開発用基本ディレクトリ構造の整備
  - `components/` (共通UIパーツ)
  - `context/` (状態管理用Context)
  - `db/` (データベースヘルパー)
  - `app/` (Expo Router 画面遷移用)

### 検証項目
- [ ] `npm run android` などの開発サーバーが正常に立ち上がること
- [ ] 依存関係エラーや型エラーが発生せずにビルド（または起動）できること
```

## Issue 2: [Phase 2] Database (SQLite) & 状態管理 (React Context) の構築

### タイトル
`[Phase 2] Database (SQLite) & 状態管理 (React Context) の構築`

### 本文
```markdown
### 概要
`expo-sqlite` を用いたローカルデータベースのテーブル定義と初期化、およびデータをアプリケーション全体でシームレスに操作するための React Context (useEvents フック) を実装します。

### タスクリスト
- [ ] `db/database.ts` にて `expo-sqlite` の接続およびテーブルの初期化ロジックを実装
  - `trips` テーブル (イベントデータ、思い出のメタデータ)
  - `items` テーブル (持ち物チェックリスト、`trip_id`による外部キー連携)
- [ ] データベースへの CRUD 抽象化関数の作成
  - `getAllEvents()`: イベント・持ち物データを結合して取得
  - `createEvent(title, date)`: 新規お出かけ予定の作成
  - `deleteEvent(id)`: イベントの削除 (Cascade による関連アイテムの自動削除)
  - `togglePackingItem(id)`: 持ち物チェック状態の反転
  - `addPackingItem(tripId, name)`: 新規持ち物の追加
- [ ] アプリの状態を保持・操作する `EventsContext` ＆ `EventsProvider` の実装 (`context/EventsContext.tsx`)
- [ ] UIから状態へ安全にアクセスできるカスタムフック `useEvents` の実装

### 検証項目
- [ ] アプリ起動時にデータベースが初期化され、エラーなく動作すること
- [ ] ダミーのイベントを追加・トグル・削除し、SQLite内のデータが意図通り更新されること
- [ ] アプリを再起動（リロード）してもデータが保持されていること
```

## Issue 5: [Phase 3-3] 思い出画面 (Memory Screen) & カメラ・フォトピッカー・保存機能

### タイトル
`[Phase 3-3] 思い出画面 (Memory Screen) & カメラ・フォトピッカー・保存機能の実装`

### 本文
```markdown
### 概要
お出かけ当日または終了後に、思い出の写真1枚と一言メモ（140文字）を記録し、アプリのローカルファイルシステムへ保存する思い出画面を実装します。

### タスクリスト
- [ ] 思い出画面 (`app/memory/[id].tsx`) の基本UI・ルーティングの実装
- [ ] `expo-image-picker` を用いた画像取得の実装
  - 「写真を撮る (カメラ)」ボタン
  - 「アルバムから選ぶ (システムフォトピッカー)」ボタン
  - ※Pixel 10 Pro (Android 14/15) の新しいフォトピッカーに適合させ、過剰なメディア権限の要求を回避
- [ ] `expo-file-system` を用いた写真データの保存
  - 画像をアプリ専用ドキュメントディレクトリ（`FileSystem.documentDirectory` 配下）に一意のファイル名で保存
  - データベース（SQLite）には **相対パス** のみを保存（iOSサンドボックス対策）
- [ ] 一言メモ入力エリアの実装（最大140文字、文字数カウント機能付き）
- [ ] 例外・エラーハンドリングの実装
  - E-1: 未保存のまま戻ろうとした際の、確認アラート（`Alert.alert`）の表示
  - E-3: ストレージ空き容量不足などで保存に失敗した際のエラーメッセージ（スナックバーやアラート）表示

### 検証項目
- [ ] カメラでの撮影、またはフォトピッカー経由で画像が取得でき、画面上にプレビュー表示されること
- [ ] メモを入力して「保存」をタップした際、画像がアプリローカルに保存され、SQLiteのパスが更新されること
- [ ] 画像やメモが変更されている状態で「保存」せずに戻るボタン（または端末の戻るジェスチャー）を操作した際、破棄確認アラートが表示されること
- [ ] 保存した画像が、アプリを再起動（リロード）しても正しく表示され続けること
```

## Issue 6: [Phase 4] エッジ・トゥ・エッジ対応、Material 3風スタイリングの洗練と検証

### タイトル
`[Phase 4] エッジ・トゥ・エッジ対応、Material 3風スタイリングの洗練と検証`

### 本文
```markdown
### 概要
Android 15 で標準となる Edge-to-Edge (画面全体描画) に準拠させ、Material 3 にインスパイアされた一貫したデザインシステム・カラーパレットを適用し、実機検証を完了させます。

### タスクリスト
- [ ] `react-native-safe-area-context` の `SafeAreaView` や `useSafeAreaInsets` フックを用いて、カメラホールやシステムナビゲーションバーとコンテンツが重ならないよう余白（インセット）を最適化
- [ ] ステータスバー、ナビゲーションバーを透過させ、エッジ・トゥ・エッジ表示に完全適合
- [ ] Material 3 風のカラーパレットを CSS Variables / TypeScript スタイルオブジェクトで定義し、アプリ全体に一貫して適用
- [ ] タップ時のネイティブ波紋（Ripple）エフェクトや、ボタンのアクティブ状態のスタイリング、画面遷移アニメーションの最適化 (Visual Polish)
- [ ] 実機 (Expo Go) を用いた徹底検証
  - 画面を回転（横画面）させたときに入力フォームの値やプレビュー画像が消失しないか（状態保持の検証）
  - 高解像度写真（大容量）を選択した際の自動的な圧縮・リサイズが正しく効いているか（メモリ負荷対策）

### 検証項目
- [ ] ステータスバーやナビゲーションバーの背面まで背景色が広がりつつ、文字やボタンは安全な領域に収まっていること
- [ ] 実機 (Expo Go) で一連の機能（イベント作成〜持ち物チェック〜写真保存・復旧）がクラッシュすることなく快適に動作すること
```
