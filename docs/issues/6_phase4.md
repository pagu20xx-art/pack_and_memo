# Issue 6: [Phase 4] エッジ・トゥ・エッジ対応、Material 3風スタイリングの洗練と検証

## 概要
Android 15 で標準となる Edge-to-Edge (画面全体描画) に準拠させ、Material 3 にインスパイアされた一貫したデザインシステム・カラーパレットを適用し、実機検証を完了させます。

## タスクリスト
- [ ] `react-native-safe-area-context` の `SafeAreaView` や `useSafeAreaInsets` フックを用いて、カメラホールやシステムナビゲーションバーとコンテンツが重ならないよう余白（インセット）を最適化
- [ ] ステータスバー、ナビゲーションバーを透過させ、エッジ・トゥ・エッジ表示に完全適合
- [ ] Material 3 風のカラーパレットを CSS Variables / TypeScript スタイルオブジェクトで定義し、アプリ全体に一貫して適用
- [ ] タップ時のネイティブ波紋（Ripple）エフェクトや、ボタンのアクティブ状態のスタイリング、画面遷移アニメーションの最適化 (Visual Polish)
- [ ] 実機 (Expo Go) を用いた徹底検証
  - 画面を回転（横画面）させたときに入力フォームの値やプレビュー画像が消失しないか（状態保持の検証）
  - 高解像度写真（大容量）を選択した際の自動的な圧縮・リサイズが正しく効いているか（メモリ負荷対策）

## 検証項目
- [ ] ステータスバーやナビゲーションバーの背面まで背景色が広がりつつ、文字やボタンは安全な領域に収まっていること
- [ ] 実機 (Expo Go) で一連の機能（イベント作成〜持ち物チェック〜写真保存・復旧）がクラッシュすることなく快適に動作すること
